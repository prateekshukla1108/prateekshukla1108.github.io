How not to make your life miserable when switching to linux

November 11, 2024 Prateek 5 min read

On this page

    Lot of people always want to switch to linux, and whenever someone wants to switch people give them the worst possible advice, almost all the blogs, youtube channels and forums tell you the stupidest things to do when they start their linux journey. So here is the Advice that not sucks.

    Decide first whether you want to switch to linux

    The best way to avoid being miserable when you switch to linux is not switching to linux. If you are accustomed to Adobe apps or Microsoft apps and don't expect to use the web versions then don't switch to linux. You will end up wasting your valuable time and at the end make yourself suffer. Only use linux if you either have everything you use in linux or you are willing to make yourself suffer and switch to native apps as well

    Expect to change your workflow

    When you are starting your linux journey, your whole workflow will be very different from what you were using before. Expect that you will not be able to do the things same way you did before. If you would try to use your old windows or mac applications like photoshop or microsoft word or the sneaky antivirus you were using before, you will inevitably fall into a whole bunch of difficult software like wine or virtualbox which will make your life super miserable

    Try to use the native applications

    Even if you have most of the applications you use in linux. You will have some packages that are not and the best way to fill the void is by using the native apps. It will be pain first but with time you will be better at it. Don't try using 'Wine' or any other emulator they are too difficult to configure and not worth wasting time.

    Do the basic homework before switching

    Make a list of all the applications you need and search them in the pakage lists of your distro. Find out about Concepts like 'Dual booting', 'Display Managers' and 'Desktop Environments'. And the most important part 'The Distro'

    The Distro Question

    Which linux distro should one use. If you look at youtube, you will get some answers like Ubuntu, PopOS, Fedora, Nobara, NixOS and in some cases Arch Linux. The thing is with most of these distributions which people regard as Noob Friendly they are actually a lot difficult to operate as they are made not to customise but settle with the default settings and when you try to tinker with some of these distributions they break. What do I mean when I say they break? they become unusable, the system either becomes non bootable or some application or function stops working.

    Use Arch based Distro

    If you are starting your linux journey I would most definitly recommand you Distribution like 'EndeavourOS' which is based on Arch linux. The reason for this recommendation is that arch based distros are usually easy to setup, their documentation is easy to find and the Arch User Repository (AUR) has almost everything you need. With arch you will face some initial pain(or maybe a lot of initial pain) but the thing is that with some help from chatgpt and forums you will be able to dodge all those difficulties. If you use anything else then firstly you will need to take a whole lot time to install everything you want, then you will have to find the documentation for your distro which will create some problems

    Always Set it on dual boot

    Dual booting is like having two different OS on one computer, allowing you to choose which one to use each time you start up. Dual booting is useful because it lets you switch back to Windows or macOS if you need specific programs that aren’t available on Linux, without uninstalling Linux or giving up your current OS. You can give linux for development purposes and windows for gaming. You get the benefits of both systems on the same device, which will decrease the 'suck' part when switching to linux
